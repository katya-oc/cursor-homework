Шпаргалка Git

  • git config --global user.name blabla - вказуємо ім'я, яке буде відображатися у полі автора для ваших коммітів
  • git config --global user.email blabal@bla - вказуємо адресу, яка відображатиметься в інфо ваших коммітів
  • git init - створюємо новий локальний репозиторій
  • git status - показує нові або змінені файли, які можна закоммітити
  • git add - додає файл в "зону очікування" комміту
  • git commit - робим комміт/зберігаємо зміну
  • git log - історія коммітів на поточній гілці
  • git checkout - переключається на вибрану гілку і робить цю гілку робочою діректорією, пересувається по коммітах
  • git branch - створює нову гілку
  • git push - загружає зміни з локального репозиторію у віддалений
  • git pull - загружає історію/зміни з віддаленого репозиторію і об'єднує з локальним
  • git merge - об'єднує гілки разом
  • git clone - скачує репозиторій з усією історією і змінами
  • git revert - відміняє зміни в комміті (комміт існує, але зміни в ньому не відбулися)
  • git reset - повністю видаляє зміни (немає ні змін ні комміту)
  • git rebase - бере ланцюг коммітів з однієї гілки і відтворює на іншій гілці
  • git cherry-pick - переносить коміт з якоїсь іншої гілки на поточну гілку
  • git stash - приховує зміни в файлах і зберігає їх окремо так, що потім можна їх вернути.

.gitignore файл необхідний для того, щоб приховувати файли і папки від Git.
